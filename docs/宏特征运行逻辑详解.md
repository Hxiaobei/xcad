# XCad 宏特征（Macro Feature）运行逻辑详解

> 基于 SOLIDWORKS `ISwComFeature` 接口，XCad 将弱类型 COM 回调封装为强类型泛型框架。

---

## 一、整体架构概览

宏特征是 SOLIDWORKS 提供的一种扩展机制，允许开发者**自定义特征类型**，这些特征可以像原生特征（如拉伸、旋转）一样存在于特征树中，支持参数化、编辑、重建等完整生命周期。

XCad 将这套机制抽象为三层架构：

```
┌──────────────────────────────────────────────────────┐
│                 用户定义的派生类                        │
│     class MyFeature : SwMacroFeatureDefinition        │
│              <TParams, TPage>                         │
└──────────────────┬───────────────────────────────────┘
                   │ 继承
┌──────────────────▼───────────────────────────────────┐
│         SwMacroFeatureDefinition (抽象基类)            │
│    ┌──────────────┐  ┌──────────────────────────┐    │
│    │ 无参数版本    │  │ SwMacroFeatureDefinition  │    │
│    │ (基础定义)    │  │ <TParams> (带参数版本)    │    │
│    └──────────────┘  └──────────┬───────────────┘    │
│                                 │ 继承                │
│                    ┌────────────▼──────────────┐     │
│                    │ SwMacroFeatureDefinition   │     │
│                    │ <TParams, TPage>           │     │
│                    │ (带UI编辑器的完整版本)      │     │
│                    └───────────────────────────┘     │
└──────────────────────────────────────────────────────┘
```

---

## 二、核心类职责划分

### 1. `SwMacroFeatureDefinition` — COM 服务器 & 生命周期入口

**文件位置**: `XCad.Sw/Sw/Features/CustomFeature/SwMacroFeatureDefinition.cs`

这是**最核心的类**，它实现了 `ISwComFeature` 接口（SOLIDWORKS COM 回调接口），充当 SOLIDWORKS 与托管代码之间的桥梁。

**三个关键 COM 回调方法：**

| COM方法 | 触发时机 | 对应 XCad 虚方法 | 返回值 |
|---------|---------|-----------------|--------|
| `Edit()` | 用户在特征树中点击"编辑特征" | `OnEditDefinition()` | `true`(成功) / `false`(失败) |
| `Regenerate()` | 模型重建(Ctrl+Q)或特征需要重新计算 | `OnRebuild()` | `true` / `false` / `string`(错误消息) / `Body`(实体) |
| `Security()` | 特征状态变化(选中、拖拽等) | `OnUpdateState()` | `swMacroFeatureSecurityOptions_e` 枚举组合 |

### 2. `SwMacroFeature<TParams>` — 特征实例（数据载体）

**文件位置**: `XCad.Sw/Sw/Features/CustomFeature/SwMacroFeature.cs`

代表特征树中**一个具体的宏特征实例**，是参数数据的容器。

关键成员：
- **`Parameters` 属性**：读取/写入参数数据，内部有 `UseCachedParameters` 缓存机制
- **`ReadParameters()`**：从 `IMacroFeatureData` (COM接口) 中提取原始数据并反序列化为 `TParams` 对象
- **`WriteParameters()`**：将 `TParams` 对象序列化回 `IMacroFeatureData`
- **`InsertComFeatureWithParameters()`**：通过 `IFeatureManager.InsertMacroFeature3()` 将特征插入特征树

### 3. `CustomFeatureParametersParser` — 参数序列化引擎

**文件位置**: `XCad.Sw/kit/CustomFeature/CustomFeatureParametersParser.cs`

这是数据在"COM原始格式"和".NET强类型对象"之间转换的核心。

**四种参数类别**（通过反射缓存分类）：

| 类别 | 识别方式 | 对应特性 | 存储方式 |
|------|---------|---------|---------|
| **Selection** | 属性类型为 `ISwSelObject` 或 `IEnumerable<ISwSelObject>` | 无额外特性 | 索引映射字符串（如 `"0,1,2"`） |
| **Dimension** | 标注 `[ParameterDimension]` 特性 | `Linear`, `Angular`, `Radial` | 独立数组（`dimTypes[]` + `dimValues[]`） |
| **EditBody** | 标注 `[ParameterEditBody]` 特性 | — | 独立数组（`editBodies[]`） |
| **Data** | 属性实现 `IConvertible`（int, double, string, enum等） | `[ParameterExclude]` 可排除 | 键值对集合 |

**两个核心方法：**

```
Parse(TParams对象)     →  ParseResult (包含 Attributes[], Selections[], DimTypes[], DimValues[], EditBodies[])
                          ↑ 方向: .NET对象 → COM数据 (写入/插入特征时使用)

BuildParameters(COM数据) → TParams对象
                          ↑ 方向: COM数据 → .NET对象 (读取/编辑特征时使用)
```

### 4. `CustomFeatureDefinitionInstanceCache` — 全局实例缓存

**文件位置**: `XCad.Sw/kit/CustomFeature/CustomFeatureDefinitionInstanceCache.cs`

全局单例缓存，确保每种宏特征定义类型只有一个实例。当 SOLIDWORKS 通过 ProgID 查找 COM 服务器时，实际上是找到这个缓存的实例。

## 三、`ISwComFeature` 三方法 与 XCad 实现的精确映射

### 3.1 Edit 方法

```csharp
// SOLIDWORKS 原生接口
System.object Edit(System.object app, System.object modelDoc, System.object feature)
// 返回值: True if editing is successful, false if not
```

XCad 实现：
```csharp
public object Edit(object app, object modelDoc, object feature) {
    var swApp = app as ISldWorks;
    var swDoc = modelDoc as IModelDoc2;
    var swFeat = feature as IFeature;     // ← 将 object 强转为 SOLIDWORKS COM 类型

    var doc = (SwDocument)App.Documents[swDoc];
    //      通过工厂方法创建 SwMacroFeature 实例
    return OnEditDefinition(App, doc, m_MacroFeatInstFact(this, swFeat, doc, (SwApplication)App));
    //     ↑ 返回 bool：true=成功  false=失败
}
```

对于带 UI 编辑器版本 `SwMacroFeatureDefinition<TParams, TPage>`：
```csharp
public override bool OnEditDefinition(...) {
    ((SwMacroFeature<TParams>)feature).UseCachedParameters = true;  // 开启参数缓存
    m_Editor.Value.Edit(doc, feature);  // 打开 PropertyManagerPage
    return true;
}
```

### 3.2 Regenerate 方法

```csharp
// SOLIDWORKS 原生接口
System.object Regenerate(System.object app, System.object modelDoc, System.object feature)
// 返回值: true / false / string(错误消息) / Body(实体)
```

XCad 实现：
```csharp
public object Regenerate(object app, object modelDoc, object feature) {
    // ... 获取文档、解析上下文 ...
    
    var res = OnRebuild(App, doc, macroFeatInst);
    //         ↑ 调用用户重写的方法, 返回 RebuildResult

    // PostRebuild 延迟处理: 注册到 OnIdleNotify 队列
    if(m_HandlePostRebuild) {
        AddDataToRebuildQueue(App, doc, macroFeatInst);
        if(!m_IsSubscribedToIdle) {
            ((SldWorks)App.Sw).OnIdleNotify += OnIdleNotify;
        }
    }

    return ParseMacroFeatureResult(res, macroFeatInst.FeatureData);
}
```

**返回值转换逻辑** (`ParseMacroFeatureResult`)：

| `RebuildResult` 类型 | 返回给 SOLIDWORKS 的 object |
|---|---|
| `null` | `null` |
| `BodyRebuildResult` (有 Bodies) | `IBody2` 或 `IBody2[]` — 官方文档说的 "Body, if a body was created" |
| `BodyRebuildResult` (无 Bodies) | `true` |
| `RebuildResult.Result == false` | `ErrorMessage` 字符串 — 官方文档说的 "String, as displayed in an error message" |
| `RebuildResult.Result == true` | `true` |

```csharp
object ParseMacroFeatureResult(RebuildResult res, IMacroFeatureData featData) {
    if(res == null) return null;

    if(!(res is BodyRebuildResult bodyRes))
        return res.Result ? (object)true : res.ErrorMessage;  // ← true 或 错误字符串

    var bodies = bodyRes.Bodies.Select(b => b.Body).ToArray();
    if(bodies.Length == 0) return true;

    // 分配面ID和边ID给SOLIDWORKS
    foreach(var body in bodies) {
        featData.GetEntitiesNeedUserId(body, out var faces, out var edges);
        AssignFaceIds(...);  // SetFaceUserId
        AssignEdgeIds(...);  // SetEdgeUserId
    }

    return bodies.Length == 1 ? (object)bodies[0] : bodies;  // ← 单个Body或Body数组
}
```

### 3.3 Security 方法

```csharp
// SOLIDWORKS 原生接口
System.object Security(System.object app, System.object modelDoc, System.object feature)
// 返回值: swMacroFeatureSecurityOptions_e 枚举组合
```

XCad 实现：
```csharp
public object Security(object app, object modelDoc, object feature) {
    var doc = (SwDocument)App.Documents[swDoc];
    return (int)OnUpdateState(App, doc, m_MacroFeatInstFact(this, swFeat, doc, (SwApplication)App));
    //     ↑ 将 CustomFeatureState_e 转为 int 返回给 SOLIDWORKS
}
```

`CustomFeatureState_e` 枚举映射到 SOLIDWORKS 的 `swMacroFeatureSecurityOptions_e`：

```csharp
public enum CustomFeatureState_e {
    Default = 0,
    CannotBeDeleted = 1,
    CannotBeSuppressed = 2,
    CannotBeReplaced = 4,
    NotEditable = 8,
    CannotBeRolledBack = 16,
    // ... 可组合使用 (Flags)
}
```

## 四、完整生命周期流程

### 4.1 注册与实例缓存

```
应用程序启动
    │
    ▼
SwAddInEx.ConnectToSW() 
    │
    ▼
SwMacroFeatureDefinition 构造函数
    ├── 调用 CustomFeatureDefinitionInstanceCache.RegisterInstance(this)
    │     └── 将实例存入静态字典 Dictionary<Type, ISwMacroFeatureDefinition>
    ├── 调用 OnConfigureServices() 配置DI服务
    ├── 创建图标文件 (通过 IIconsCreator)
    └── 完成注册
```

`CustomFeatureDefinitionInstanceCache` 是一个**全局单例缓存**，确保每种宏特征定义类型只有一个实例。当 SOLIDWORKS 通过 ProgID 查找 COM 服务器时，实际上是找到这个缓存的实例。

### 4.2 插入特征

```
用户调用 definition.Insert(doc, data)
    │
    ▼
SwMacroFeatureDefinition<TParams,TPage>.Insert()
    │
    ▼
SwMacroFeatureEditor.Insert(doc, data)
    │
    ▼
创建 SwMacroFeature<TParams> 实例 (未提交状态)
    │
    ▼
ConvertParamsToPage() → TPage (页面数据)
    │
    ▼
显示 PropertyManagerPage, 用户编辑参数
    │
    ▼ (每次页面变更)
ConvertPageToParams() → TParams (新参数)
    │
    ▼ (如果ShouldUpdatePreview返回true)
CreatePreviewGeometry() → 预览几何体
    │
    ▼ (用户点击确定)
OnFeatureInserting()
    │
    ▼
feat.Commit() (提交特征)
    │
    ▼
SwMacroFeature<TParams>.InsertComFeatureWithParameters()
    ├── m_ParamsParser.Parse(Parameters) → ParseResult
    ├── SeparateParameters() → paramNames[], paramTypes[], paramValues[]
    └── InsertComFeatureBase() 
         └── m_FeatMgr.InsertMacroFeature3(baseName, progId, ...)
              └── SOLIDWORKS 在特征树中创建特征
```

### 4.3 重建（Regeneration）— 最关键的流程

这是用户按 `Ctrl+Q` 或模型自动重建时的流程：

```
SOLIDWORKS 调用 ISwComFeature.Regenerate(app, modelDoc, feature)
    │
    ▼
SwMacroFeatureDefinition.Regenerate()
    ├── 1. 检查 Provider 信息 (Sw2016+)
    ├── 2. 获取文档指针
    ├── 3. 解析上下文文档 (装配体中的编辑组件)
    ├── 4. 通过 m_MacroFeatInstFact 创建 SwMacroFeature<TParams> 实例
    │
    ▼
调用 OnRebuild(app, doc, macroFeatInst) 
    │  └── 虚方法, 由派生类 SwMacroFeatureDefinition<TParams> 重写
    │
    ▼
SwMacroFeatureDefinition<TParams>.OnRebuild()
    ├── paramsFeat.UseCachedParameters = true  ← 启用缓存避免重复读取COM
    ├── paramsFeat.ReadParameters() 读取参数
    │     ├── ExtractRawParameters() → 从IMacroFeatureData读取原始数据
    │     │     ├── GetParameters() → 基础参数 (int/double/string)
    │     │     ├── GetSelections3() → 选择对象 + 变换矩阵
    │     │     ├── GetDisplayDimensions() → 显示尺寸
    │     │     └── EditBodies → 编辑主体
    │     ├── ConvertParameters() → 版本兼容转换
    │     └── BuildParameters() → 构建 TParams 对象
    │
    ├── 调用用户重写的 OnRebuild(app, doc, paramsFeat) 
    │     └── 用户通过 paramsFeat.Parameters 获取参数
    │     └── 计算几何体, 返回 RebuildResult
    │
    ├── 处理尺寸对齐 (AlignDimension 或 AssignDimension)
    │
    ├── 处理 PostRebuild 延迟队列
    │     └── 如果订阅了 PostRebuild 事件
    │         └── AddDataToRebuildQueue() 保存数据
    │         └── 订阅 OnIdleNotify (SOLIDWORKS空闲回调)
    │
    └── 返回结果给 SOLIDWORKS (ParseMacroFeatureResult)
          ├── 如果是 BodyRebuildResult → 处理实体
          │     ├── AssignFaceIds() 分配面ID
          │     └── AssignEdgeIds() 分配边ID
          └── 如果是状态结果 → 返回 true/error message
```

### 4.4 延迟 PostRebuild

```
SOLIDWORKS 进入空闲状态
    │
    ▼
OnIdleNotify() 被触发
    ├── 取消订阅 (只触发一次)
    ├── 遍历 m_RebuildFeaturesQueue
    │     └── DispatchPostBuildData(data)
    │           └── 触发 PostRebuild 事件
    │                 └── 用户可以在重建完成后执行额外操作
    └── 清空队列
```

### 4.5 编辑特征

```
用户在特征树中右键 → "编辑特征"
    │
    ▼
SOLIDWORKS 调用 ISwComFeature.Edit(app, modelDoc, feature)
    │
    ▼
SwMacroFeatureDefinition.Edit()
    ├── 获取文档对象
    ├── 创建 SwMacroFeature 实例
    └── 调用 OnEditDefinition()
    │
    ▼
SwMacroFeatureDefinition<TParams, TPage>.OnEditDefinition()
    ├── 开启 UseCachedParameters (避免每次读取COM)
    ├── m_Editor.Value.Edit(doc, feature)
    │     └── 打开 PropertyManagerPage
    │     └── 用户编辑 → ConvertPageToParams → 预览更新
    │
    └── 用户点击确定
          ├── CompleteFeature()
          ├── ApplyParametersCache() → 将缓存参数写入COM
          └── UseCachedParameters = false
```

## 五、参数数据流详解

### 5.1 写入方向：TParams → COM

```
TParams 对象
    │
    ▼
CustomFeatureParametersParser.Parse()
    ├── TraverseParametersDefinition() 遍历属性 (使用反射缓存)
    │     ├── Selection属性 → 收集到 selectionList, 生成索引映射参数
    │     ├── Dimension属性 → 收集到 dimsList (类型+值)
    │     ├── EditBody属性 → 收集到 editBodiesList, 生成索引映射参数
    │     └── Data属性 → 收集到 paramAttsList (键值对)
    │
    ├── 处理版本特性 [ParametersVersion]
    │     └── 添加 __dimsVersion 和 __paramsVersion 参数
    │
    └── 返回 ParseResult
          ├── Attributes[] → 所有键值对参数
          ├── Selections[] → ISwSelObject数组
          ├── DimTypes[] + DimValues[]
          └── EditBodies[]
```

### 5.2 读取方向：COM → TParams

```
IMacroFeatureData (COM接口)
    │
    ▼
SwMacroFeature.ExtractRawParameters()
    ├── featData.GetParameters() → paramNames[], paramTypes[], paramValues[]
    ├── featData.GetSelections3() → 选择对象 + 变换矩阵
    ├── featData.GetDisplayDimensions() → 显示尺寸
    └── featData.EditBodies → 编辑主体
    │
    ▼
CustomFeatureParametersParser.ConvertParameters() (版本兼容)
    │
    ▼
CustomFeatureParametersParser.BuildParameters()
    ├── 创建 TParams 实例 (Activator.CreateInstance)
    ├── TraverseParametersDefinition()
    │     ├── Selection → AssignObjectsToProperty() (通过索引映射恢复)
    │     ├── Dimension → 从尺寸数组读取值
    │     ├── EditBody → AssignObjectsToProperty()
    │     └── Data → TryGetParameterValue() 读取值并类型转换
    │
    └── 返回 TParams 对象
```

## 六、关键设计模式与优化

| 模式/优化 | 实现位置 | 说明 |
|-----------|---------|------|
| **反射元数据缓存** | `CustomFeatureParametersParser.GetOrBuildMetadata()` | `ConcurrentDictionary<Type, PropertyMetadata[]>` 避免重复反射 |
| **单例解析器** | `CustomFeatureParametersParser.Instance` | 全局唯一实例 |
| **全局定义缓存** | `CustomFeatureDefinitionInstanceCache` | `Dictionary<Type, ISwMacroFeatureDefinition>` 每种类型只实例化一次 |
| **索引映射 O(n)** | `AddParametersForObjects()` | 用 Dictionary 做索引映射，替代 O(n²) 的 IndexOf |
| **FaultObject 占位** | `FaultObjectFactory` | 实体丢失时创建占位对象而非抛异常 |
| **延迟 PostRebuild** | `OnIdleNotify` 队列 | 非阻塞方式在 SOLIDWORKS 空闲时执行后处理 |
| **参数缓存** | `UseCachedParameters` / `m_ParametersCache` | 重建期间避免重复读取 COM 接口 |
| **实体变换缓存** | `m_EntitiesTransformsCache` | 缓存选择对象的变换矩阵 |
| **TryParse 容错** | `GetObjectIndices()` | 使用 `int.TryParse` 防止损坏数据导致崩溃 |

## 七、三层调用链总结

```
SOLIDWORKS 引擎
    │  调用 COM 接口
    ▼
ISwComFeature (COM Interface)
    ├── Edit(object app, object modelDoc, object feature) → object
    ├── Regenerate(object app, object modelDoc, object feature) → object
    └── Security(object app, object modelDoc, object feature) → object
    │
    │  XCad: SwMacroFeatureDefinition 实现此接口
    │  将 object 参数强转为 SOLIDWORKS COM 类型
    │  将返回值从 .NET 类型转换为 object
    ▼
XCad 虚方法 (用户可重写)
    ├── OnEditDefinition(ISwApplication, ISwDocument, ISwMacroFeature) → bool
    ├── OnRebuild(ISwApplication, ISwDocument, ISwMacroFeature<TParams>) → RebuildResult
    └── OnUpdateState(ISwApplication, ISwDocument, ISwMacroFeature) → CustomFeatureState_e
    │
    │  用户派生类重写这些方法，实现业务逻辑
    ▼
最终用户代码
    class MyMacroFeature : SwMacroFeatureDefinition<MyParams, MyPage> {
        override OnRebuild(...) { /* 计算几何 */ }
        override ConvertPageToParams(...) { /* UI→数据 */ }
    }
```

---

> XCad 的核心价值：**将 SOLIDWORKS 弱类型 `object` 接口封装为强类型泛型接口**，通过 `CustomFeatureParametersParser` 自动完成 COM 数据与 .NET 对象的双向序列化，让开发者只需关注业务逻辑。
